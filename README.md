## ⭐ SIDT — Sistema Integral de Diagnóstico Tecnomotriz

<div align="center">

[![Tipo](https://img.shields.io/badge/Tipo-Sistema_Empresarial-1F6FEB?style=for-the-badge)](https://github.com/IngKevin710)
[![Estado](https://img.shields.io/badge/Estado-En_Producci%C3%B3n-00D4FF?style=for-the-badge)](https://github.com/IngKevin710)
[![Cliente](https://img.shields.io/badge/Cliente-Tecnomotos_S.A.S-0D1117?style=for-the-badge)](https://github.com/IngKevin710)
[![Rol](https://img.shields.io/badge/Rol-Full_Stack_Developer-1F6FEB?style=for-the-badge)](https://github.com/IngKevin710)

</div>

> Plataforma web empresarial que digitaliza por completo las operaciones de un Centro de Diagnóstico Automotor: desde la gestión de citas y clientes hasta el inventario de maquinaria con detección automática de EPP mediante visión artificial. **+50 tablas relacionales · 7 módulos · 5 roles diferenciados.**

<br/>

### 🏛️ Arquitectura

```
╔══════════════════════════════════════════════════════════════════════╗
║                      🌐  FRONTEND (React + Tailwind)                 ║
║              Componentes por rol · Auth JWT · Axios                  ║
╠══════════════════════════════════════════════════════════════════════╣
║                       🛡️  API Gateway / Nginx                        ║
╠══════════════════════════════════════╦═══════════════════════════════╣
║   ⚙️  Spring Boot 3 (Java 21)         ║  🤖 FastAPI + YOLOv8          ║
║      RESTful API · JWT · JPA          ║     Detección de EPP           ║
╠══════════════════════════════════════╩═══════════════════════════════╣
║   🐘 PostgreSQL 16          │          📦 MinIO (Object Storage)     ║
║   +50 tablas relacionales    │          Compatible con S3              ║
╠══════════════════════════════════════════════════════════════════════╣
║         🐳 Docker Compose · dev → staging → production               ║
╚══════════════════════════════════════════════════════════════════════╝
```

### 📦 Módulos del Sistema

| | Módulo | Descripción | Stack |
|---|--------|-------------|-------|
| 🔐 | **Autenticación** | JWT, 5 roles, blacklist de tokens, recuperación de contraseña | `Spring Security` `JWT` `BCrypt` |
| 📅 | **Gestión de Citas** | CRUD de citas para SOAT y revisión técnico-mecánica con recordatorios | `Spring Scheduler` `MailHog` `Cron` |
| 👥 | **Control de Personal** | Asistencia, permisos, quejas, sanciones, evaluaciones | `JPA` `DTOs` `Validation` |
| 🔧 | **Inventario de Maquinaria** | Mantenimientos, calibraciones, alertas de vencimiento | `Scheduler` `Audit Trail` |
| 🛡️ | **Detección de EPP (IA)** | Verificación automática de uso de EPP mediante visión artificial | `Python` `FastAPI` `YOLOv8` `MinIO` |
| 💰 | **Simulador de Crédito** | Cálculo de financiamiento para servicios del CDA | `React` `Business Logic` |
| 🔔 | **Notificaciones** | Alertas automáticas: vencimientos, recordatorios, permisos | `Schedulers` `Email Service` |

<div align="center">

> Aplicación de gestión desarrollada en equipo, con sincronización en tiempo real mediante **Firebase Realtime Database** y autenticación gestionada por **Firebase Auth**. Proyecto enfocado en la práctica de **bases de datos NoSQL**, trabajo colaborativo con Git y arquitectura basada en componentes reutilizables.

<br/>

<table>
<tr>
<td width="50%" valign="top">

#### 💡 Lo que aprendí

- Modelado de datos en **NoSQL** (estructura jerárquica vs. relacional)
- Sincronización en tiempo real con **listeners de Firebase**
- Autenticación con providers de **Firebase Auth**
- Reglas de seguridad de Firestore para control de acceso
- Trabajo colaborativo con **branches de Git** y resolución de conflictos
- Despliegue continuo con **Firebase Hosting**

</td>
<td width="50%" valign="top">

#### 🔧 Stack Técnico

```diff
+ React (Hooks + Context API)
+ Firebase Realtime Database
+ Firebase Authentication

+ CSS3 + diseño responsive
+ Git Flow para colaboración
```
